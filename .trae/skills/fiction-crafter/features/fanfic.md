# 番外/同人/仿写支持

Fiction Crafter 支持多种特殊创作模式，满足不同的创作需求。

---

## 番外创作

### 什么是番外
番外是主线故事之外的补充内容，通常：
- 发生在主线时间线之前/之后
- 聚焦配角的独立故事
- 补充主线未交代的情节
- 满足读者的衍生需求

### 番外类型

#### 类型1：前传
**时间点**：主线故事之前
**内容**：主角的过去、起源故事、前因后果
**示例**：
- 主角如何获得金手指
- 主角的童年经历
- 重要配角的过去

#### 类型2：后传
**时间点**：主线故事之后
**内容**：结局后的生活、后代故事、世界变化
**示例**：
- 主角退休后的生活
- 下一代的成长
- 世界的后续发展

#### 类型3：支线
**时间点**：主线故事期间
**内容**：主线未详述的支线剧情
**示例**：
- 配角视角的故事
- 某个事件的详细过程
- 平行发生的故事

#### 类型4：IF线
**时间点**：主线某个节点
**内容**：如果做出不同选择会怎样
**示例**：
- 如果主角没有获得金手指
- 如果主角选择了另一条路
- 如果某个角色没有死

#### 类型5：日常/狗粮
**时间点**：任意
**内容**：轻松日常、感情互动、甜蜜片段
**示例**：
- 主角和CP的日常生活
- 角色们的互动趣事
- 节日特别篇

### 番外创作流程

```
选择番外类型
    ↓
确定时间点
    ↓
选择角色
    ↓
设计情节
    ↓
保持人设一致性
    ↓
创作
    ↓
审核（标准流程）
```

### 番外规划模板

```markdown
# 《[书名]》番外规划

## 基本信息
- **番外名称**: [名称]
- **类型**: [前传/后传/支线/IF线/日常]
- **时间点**: [在主线中的位置]
- **涉及角色**: [角色列表]
- **预计字数**: [字数]

## 内容概要
[一句话概括]

## 与主线的关系
- **补充信息**: [补充了什么]
- **伏笔回收**: [回收了哪些伏笔]
- **人物深化**: [深化了哪些人物]

## 创作要点
- [要点1]
- [要点2]

## 注意事项
- 保持人设一致
- 不破坏主线设定
- 时间线不矛盾
```

## 同人创作

### 什么是同人
基于已有作品（小说、动漫、游戏等）进行的二次创作。

### 同人类型

#### 类型1：原著向
**特点**：严格遵循原作设定
**内容**：补充原作未写的情节、配角视角
**注意**：不改动原作主线

#### 类型2：AU（Alternative Universe）
**特点**：改变原作背景设定
**示例**：
- 现代AU：古代角色放到现代
- 校园AU：所有角色都在学校
- 职场AU：角色们成为同事
- 奇幻AU：普通作品加入奇幻元素

#### 类型3： crossover（ crossover）
**特点**：多个作品的角色相遇
**示例**：
- 不同小说的主角相遇
- 动漫角色穿越到小说世界
- 游戏角色进入现实世界

#### 类型4：性转/年龄操作
**特点**：改变角色的性别或年龄
**示例**：
- 角色性别转换
- 角色年龄变大/变小
- 角色身份互换

#### 类型5：CP向
**特点**：聚焦特定角色配对
**示例**：
- 原作CP的甜蜜日常
- 拉郎配（非官配）
- 感情线发展

### 同人创作原则

#### 版权规避
- 不用于商业用途
- 不侵犯原作者权益
- 标注原作信息
- 不抄袭原作大段内容

#### 人设保持
- 保持角色核心性格
- 保持角色能力设定
- 保持角色关系基础
- 允许合理发展

#### 创新性
- 有自己的创意
- 不只是复述原作
- 提供新的视角
- 探索新的可能性

### 同人创作流程

```
选择原作
    ↓
分析原作设定
    ↓
确定同人类型
    ↓
设计差异化内容
    ↓
创作
    ↓
标注原作信息
```

## 仿写创作

### 什么是仿写
模仿特定作者或作品的风格进行创作。

### 仿写类型

#### 类型1：作者风格仿写
**示例**：
- 金庸风格：武侠、历史感、人物众多
- 古龙风格：悬疑、短句、意境
- 南派三叔风格：盗墓、悬疑、恐怖
- 天蚕土豆风格：升级、热血、爽文

#### 类型2：作品风格仿写
**示例**：
- 《斗破苍穹》风格：退婚流、升级流
- 《鬼吹灯》风格：盗墓、探险、民俗
- 《全职高手》风格：电竞、群像、热血
- 《庆余年》风格：权谋、幽默、反转

#### 类型3：时代风格仿写
**示例**：
- 古典白话风格
- 民国文风
- 现代网络文风
- 翻译腔风格

### 仿写要点

#### 风格分析
1. **语言特点**：句式、词汇、修辞
2. **叙事方式**：视角、节奏、结构
3. **人物塑造**：类型、发展、关系
4. **情节设计**：套路、反转、高潮
5. **世界观**：设定、规则、细节

#### 保持原创
- 模仿风格，不抄袭内容
- 有自己的故事
- 有自己的角色
- 只是风格相似

### 仿写创作流程

```
选择仿写对象
    ↓
深度分析风格
    ↓
提取风格特征
    ↓
设计原创故事
    ↓
用仿写风格创作
    ↓
检查原创性
```

## 特殊创作模式使用

### 在 Skill 中使用

用户可以通过以下方式触发特殊创作模式：

```
用户：写个番外，讲讲主角获得系统之前的故事
    ↓
Agent识别为"前传番外"
    ↓
使用番外创作流程
    ↓
保持人设一致
    ↓
创作番外

用户：写个同人，让斗破的主角穿越到斗罗大陆
    ↓
Agent识别为"crossover同人"
    ↓
分析两个原作设定
    ↓
设计穿越情节
    ↓
创作同人

用户：仿写金庸风格，写一个武侠故事
    ↓
Agent识别为"作者风格仿写"
    ↓
分析金庸风格特点
    ↓
设计原创武侠故事
    ↓
用金庸风格创作
```

### 配置选项

在 `.learnings/config.md` 中配置：

```yaml
special_modes:
  fanfic:
    enabled: true
    max_word_count: 5000
    require_canon_consistency: true
    
  fanfiction:
    enabled: true
    require_original_attribution: true
    non_commercial_only: true
    
  imitation:
    enabled: true
    originality_check: true
    style_analysis_depth: "detailed"
```
