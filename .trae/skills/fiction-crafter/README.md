# Fiction Crafter v3.0

专业级 AI 小说创作工具，采用 **Planner → Writer → Auditor → Reviser → Gatekeeper** 六 Agent 协作架构，支持写、审、改全流程自动化。

> **v3.0 重大更新**：完整实现 InkOS 风格的多 Agent 协作架构，新增 Planner（规划）、Critique（批评）、Gatekeeper（门控）三大 Agent，支持番外/同人/仿写、市场雷达、数据分析等高级功能。

## 硬性规则

1. **📏 篇幅规划**：创作前必须询问用户篇幅需求，根据题材给出专业建议，避免题材与篇幅不匹配
2. **🔍 书名查重**：确定小说名字前，必须先在各平台（特别是番茄小说）查重，避免重名
3. **📁 文件夹规则**：每部小说在当前目录创建独立文件夹，便于管理
4. **📋 大纲优先**：必须先形成完整大纲，经用户确认后才能开始创作
5. **🎨 推广素材（HTML）**：大纲确认后必须生成 HTML 形式的封面图和推文，方便下载
6. **📚 全卷规划**：必须完成所有卷（3-5卷）的详细规划，方便审查
7. **📄 章节独立**：每个章节保存为独立的 Markdown 文件
8. **📝 字数限制**：每章字数限制在 2000-3000 字，确保内容连贯
9. **⚖️ 价值观审核**：所有内容必须符合社会主义核心价值观，历史题材不得恶意篡改历史，不得出现污秽、误导等不良内容
10. **📑 文档化要求**：故事大纲、每一卷规划、全卷规划总览等所有规划文档必须保存为 Markdown 文件，不得仅口头描述
11. **🔒 严格执行大纲**：一旦总体大纲、每一卷、每一章的大纲确定下来，在创作时必须严格按照大纲进行，不得随意调整。如需调整，必须通过 Gatekeeper 门控，经用户确认后方可修改
12. **📛 文件名规范**：章节文件名必须是 `第XXX章 章节名.md` 格式，例如：`第1章 初入传承.md`、`第15章 突破境界.md`
13. **🍅 番茄签约标准**：所创作内容必须符合番茄小说签约标准，帮助用户成功签约
14. **📝 最低字数要求**：每章内容不得少于1000字，确保内容充实完整

## 核心功能

### 1. 多 Agent 协作创作

- **Planner（规划师）**：全局规划、题材分析、世界观构建、人物设计、大纲规划
- **Writer（写作专员）**：章节创作、风格适配、节奏把控
- **Auditor（审核专员）**：15维度深度审核、多模型交叉验证、专业化审核
- **Reviser（修改专员）**：精准修复、优点保留、修改记录
- **Critique（批评家）**：文学性分析、可读性评估、市场适配
- **Gatekeeper（门控官）**：人工审核门控、流程控制

### 2. 多题材支持

支持玄幻、修仙、都市、科幻、悬疑、言情、历史等多种题材，每种题材都有专门的创作规范和审核标准。

### 3. 特殊创作模式

- **番外创作**：前传、后传、支线、IF线、日常/狗粮
- **同人创作**：原著向、AU、Crossover、性转/年龄操作、CP向
- **仿写创作**：作者风格仿写、作品风格仿写、时代风格仿写

### 4. 市场雷达与数据分析

- 热门题材监测
- 读者偏好分析
- 竞品分析
- 作品数据追踪
- 质量趋势分析
- 健康度评估

### 5. 人工审核门控

- **必停节点**：大纲确认、卷级规划、严重问题、循环超限
- **选停节点**：提示词、章节初稿、关键章节、Critique反馈
- 智能建议与流程控制

## 快速开始

### 使用方法

在对话中输入：

```
帮我写一个都市修仙的爽文，主角获得了一个签到系统
```

系统将自动：
1. 分析需求并生成提示词
2. 构建世界观和人物设定
3. 制定全局大纲和卷级规划
4. **等待你确认后进入创作阶段（大纲具有约束力）**
5. **严格按照确认后的大纲进行创作（规则6）**
6. 每章创作完成后自动审核（含大纲执行检查）
7. 审核通过则发布，不通过则修改
8. 关键节点暂停等待你的决策
9. 如需调整大纲，必须通过门控申请

> **重要提示**：一旦大纲经你确认，系统会严格按照大纲创作，不会随意调整。如需修改大纲，系统会暂停并询问你的意见。

### 特殊创作模式

```
写个番外，讲讲主角获得系统之前的故事
写个同人，让斗破的主角穿越到斗罗大陆
仿写金庸风格，写一个武侠故事
```
