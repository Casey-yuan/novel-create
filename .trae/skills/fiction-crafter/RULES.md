# Fiction Crafter 创作规则

本文档定义了 Fiction Crafter 在进行小说创作时必须遵守的硬性规则。

---

## 规则 1：文件夹组织规则

### 强制要求
**每部小说必须在当前工作目录下创建独立的文件夹**

### 文件夹结构

```
当前工作目录/
└── [小说名]/                          # 小说主文件夹
    ├── README.md                      # 小说基本信息
    ├── outline/                       # 大纲文件夹
    │   ├── 全局大纲.md                 # 完整故事大纲
    │   ├── 人物设定.md                 # 人物档案
    │   ├── 世界观.md                   # 世界观设定
    │   └── 卷级规划/                   # 各卷详细规划
    │       ├── 第一卷.md
    │       ├── 第二卷.md
    │       └── ...
    ├── chapters/                      # 章节文件夹
    │   ├── 第一卷/
    │   │   ├── 第001章.md
    │   │   ├── 第002章.md
    │   │   └── ...
    │   ├── 第二卷/
    │   │   └── ...
    │   └── ...
    ├── assets/                        # 资源文件夹
    │   ├── cover.png                  # 小说封面
    │   ├── 推文/                      # 推广素材
    │   │   ├── 推文1.md
    │   │   └── 推文2.md
    │   └── images/                    # 插图
    ├── .learnings/                    # 记忆系统
    │   ├── CHARACTERS.md
    │   ├── LOCATIONS.md
    │   ├── PLOT_POINTS.md
    │   └── ...
    └── config/                        # 小说专属配置
        └── settings.yaml
```

### 实施步骤

1. **接收创作请求时**
   - 询问或确定小说名称
   - 检查当前目录是否已存在同名文件夹
   - 如不存在，创建文件夹结构
   - 如存在，询问是继续创作还是覆盖

2. **文件夹命名规范**
   - 使用小说原名（中文）
   - 避免特殊字符
   - 示例：`《斗破苍穹》/`、`《诡秘之主》/`

## 规则 2：大纲优先规则

### 强制要求
**必须先形成完整的故事大纲，经用户确认后才能开始章节创作**

### 大纲阶段流程

```
用户需求
    ↓
生成提示词（Prompt）
    ↓
构建世界观设定
    ↓
设计人物档案
    ↓
制定全局大纲（3-5卷）
    ↓
【Gatekeeper】大纲确认门控
    ↓
用户确认/修改
    ↓
制定第一卷详细规划
    ↓
【Gatekeeper】卷级规划确认
    ↓
用户确认后 → 开始创作
```

### 大纲内容要求

#### 全局大纲必须包含
1. **故事梗概**（300-500字）
2. **核心主题**（1-3个关键词）
3. **主线剧情**（起承转合）
4. **分卷规划**（每卷目标、高潮、结局）
5. **关键转折点**（至少3个）
6. **伏笔设计**（前期埋设、后期回收）

#### 卷级规划必须包含
1. **本卷目标**（主角要达成什么）
2. **章节分解表**（每章任务、爽点、字数）
3. **高潮设计**（位置、内容、爽点）
4. **人物出场**（新人物、人物发展）
5. **世界展开**（新地图、新设定）

### 用户确认机制

- **大纲确认**：展示完整大纲，等待用户确认或修改
- **卷级规划确认**：每卷开始前展示详细规划
- **修改支持**：用户可随时要求调整大纲

## 规则 3：推广素材生成规则

### 强制要求
**大纲确认后，必须生成小说封面图和推广推文**

### 封面图生成

#### 要求
- 尺寸：建议 600x800px（小说封面标准比例 3:4）
- 格式：PNG
- 保存位置：`[小说名]/assets/cover.png`

#### 封面元素
1. **书名**：醒目、有设计感
2. **副标题**（可选）：一句话卖点
3. **视觉元素**：符合题材风格
   - 玄幻：神秘、宏大
   - 修仙：仙气、意境
   - 都市：现代、时尚
   - 科幻：科技、未来
   - 悬疑：神秘、紧张
   - 言情：唯美、浪漫

#### 生成方式
使用 AI 图像生成工具，根据以下 Prompt 模板：

```
小说封面设计
书名：[书名]
题材：[题材]
风格：[风格描述]
元素：[关键视觉元素]
色调：[主色调]
要求：
- 600x800px 竖版
- 书名醒目
- 符合网文封面审美
- 有视觉冲击力
```

### 推广推文生成

#### 要求
- 数量：至少3条不同风格的推文
- 保存位置：`[小说名]/assets/推文/`
- 格式：Markdown

#### 推文类型

**类型1：悬念型**
- 突出核心悬念
- 引发读者好奇
- 适合悬疑、玄幻

**类型2：爽点型**
- 突出爽文元素
- 强调打脸、升级
- 适合爽文、系统流

**类型3：情感型**
- 突出人物情感
- 引发共鸣
- 适合言情、虐文

**类型4：设定型**
- 突出世界观设定
- 强调独特性
- 适合科幻、奇幻

#### 推文模板

```markdown
# 《[书名]》推广推文

## 推文1：[类型]
[推文内容，100-200字]

## 推文2：[类型]
[推文内容，100-200字]

## 推文3：[类型]
[推文内容，100-200字]

## 标签
#[标签1] #[标签2] #[标签3]

## 适用平台
- 微博
- 小红书
- 抖音文案
- 知乎
```

## 规则 4：章节文件组织规则

### 强制要求
**每个章节必须保存为独立的 Markdown 文件，禁止所有章节写在一个文件中**

### 文件命名规范

```
第[卷号]卷/
└── 第[章节号]章[章节标题].md

示例：
第一卷/
├── 第001章退婚之辱.md
├── 第002章神秘戒指.md
├── 第003章三年之约.md
└── ...
```

### 文件命名规则

1. **章节序号**
   - 格式：三位数字，不足补零
   - 示例：`001`, `023`, `156`

2. **章节标题**
   - 简洁有力，5-15字
   - 体现本章核心内容或爽点
   - 示例：`退婚之辱`、`秘境探险`、`突破境界`

3. **完整文件名**
   - 格式：`第[序号]章[标题].md`
   - 示例：`第001章退婚之辱.md`

### 章节文件内容格式

```markdown
# 第[序号]章 [标题]

## 章节信息
- **所属卷**: 第[X]卷 [卷名]
- **章节序号**: 第[X]章
- **预计字数**: XXXX字
- **实际字数**: XXXX字
- **审核评分**: XX分
- **创作时间**: YYYY-MM-DD

## 本章任务
- [任务1]
- [任务2]

## 正文

[章节正文内容]

---

## 审核记录
[如有审核记录，在此记录]

## 修改记录
[如有修改记录，在此记录]
```

### 实施步骤

1. **创作前**
   - 检查章节文件夹是否存在
   - 如不存在，创建 `chapters/第[X]卷/` 文件夹

2. **创作时**
   - 按模板生成章节内容
   - 统计字数
   - 保存为独立文件

3. **审核后**
   - 更新审核评分
   - 如有修改，更新修改记录

4. **章节管理**
   - 每卷一个子文件夹
   - 文件列表即为章节列表
   - 便于定位和修改

## 规则 5：文档化要求规则

### 强制要求
**所有规划文档必须保存为 Markdown 文件，禁止仅口头描述**

### 必须文档化的内容

#### 1. 全局大纲（必须）
- **文件路径**：`outline/全局大纲.md`
- **内容要求**：
  - 故事梗概（300-500字）
  - 核心主题（1-3个关键词）
  - 主线剧情（起承转合）
  - 分卷规划（每卷目标、高潮、结局）
  - 关键转折点（至少3个）
  - 伏笔设计（前期埋设、后期回收）
- **禁止**：仅口头描述大纲，不形成文档

#### 2. 全卷规划总览（必须）
- **文件路径**：`outline/全卷规划总览.md`
- **内容要求**：
  - 全卷概览表（各卷核心目标、章节范围、爽点类型）
  - 主角成长线（贯穿全卷的成长轨迹）
  - 关键剧情节点（全卷重要剧情点分布）
  - 伏笔规划（长期伏笔和卷间伏笔设计）
  - 人物出场规划（主要人物出场卷和人物弧光）
  - 世界展开规划（新地图、新势力、新设定展开计划）
- **禁止**：仅口头描述规划，不形成文档

#### 3. 卷级规划（每卷必须）
- **文件路径**：`outline/卷级规划/第X卷.md`
- **内容要求**：
  - 本卷目标（主角要达成什么）
  - 章节分解表（每章任务、爽点、字数限制）
  - 爽点分布图（可视化爽点分布）
  - 高潮节点（中高潮、卷终高潮设计）
  - 新元素引入（新角色、新地点、新设定）
  - 伏笔规划（本卷埋设和回收的伏笔）
- **禁止**：仅口头描述卷级规划，不形成文档

#### 4. 人物档案（必须）
- **文件路径**：`outline/人物档案.md`
- **内容要求**：
  - 主角详细人设
  - 重要配角档案
  - 人物关系图
  - 人物成长轨迹
- **禁止**：仅口头描述人物，不形成文档

#### 5. 世界观设定（必须）
- **文件路径**：`outline/世界观设定.md`
- **内容要求**：
  - 时代背景
  - 力量体系/规则体系
  - 势力分布
  - 重要地点
- **禁止**：仅口头描述世界观，不形成文档

### 文档格式要求

#### 文件格式
- **格式**：Markdown (.md)
- **编码**：UTF-8
- **换行**：LF 或 CRLF

#### 文档结构
```markdown
# 文档标题

## 基本信息
- **创建时间**：YYYY-MM-DD
- **最后更新**：YYYY-MM-DD
- **版本**：v1.0

## 正文内容
[详细内容]

## 更新记录
- YYYY-MM-DD: [更新内容]
```

#### 规划阶段检查

- [ ] `outline/人物档案.md` 已创建并保存
- [ ] `outline/世界观设定.md` 已创建并保存
- [ ] 所有文档均为 Markdown 格式
- [ ] 所有文档内容完整，非空文件

#### 创作阶段检查
- [ ] 每章创作前，相关规划文档已存在
- [ ] 如需调整规划，已更新对应文档

## 规则 6：严格执行大纲规则

### 强制要求
**一旦总体大纲、每一卷、每一章的大纲确定下来，在创作时必须严格按照大纲进行，不得随意调整**

### 大纲的权威性

#### 确定后的大纲具有约束力
- **总体大纲**：确定后，故事主线、核心设定、人物关系不得随意更改
- **卷级规划**：确定后，本卷目标、章节数、高潮节点不得随意更改
- **章节规划**：确定后，本章任务、爽点类型、关键情节不得随意更改

#### 创作时的遵循原则
1. **核心任务必须完成**：规划中规定的本章核心任务必须完成
2. **爽点类型必须一致**：按照规划呈现指定的爽点类型（打脸/升级/收获/反转/情感）
3. **情绪曲线必须符合**：遵循规划中设计的情绪曲线（起-承-转-合）
4. **关键情节必须发生**：规划中设定的关键情节必须发生，不得跳过
5. **人物状态必须连贯**：角色状态变化必须符合规划

### 禁止行为

#### 绝对禁止（无需门控，直接拒绝）
- ❌ 未经用户同意，擅自改变故事主线
- ❌ 未经用户同意，擅自增减主要人物
- ❌ 未经用户同意，擅自改变核心设定
- ❌ 擅自跳过规划中设定的关键情节
- ❌ 擅自改变章节规划中规定的核心任务

#### 需要门控（重大调整需用户确认）
- ⚠️ 发现规划不合理，需要调整后续大纲
- ⚠️ 创作过程中发现更好的剧情走向
- ⚠️ 需要增加/删减章节
- ⚠️ 需要调整爽点分布
- ⚠️ 需要改变某章的核心任务

### 调整大纲的正规流程

```
创作中发现需要调整大纲
    ↓
暂停当前创作
    ↓
【Gatekeeper】大纲调整门控
    → 说明调整原因
    → 展示当前规划
    → 展示建议调整方案
    → 对比调整前后的影响
    ↓
等待用户决策
    → 同意调整：更新大纲文档，继续创作
    → 拒绝调整：按原大纲继续创作
    → 部分调整：按用户意见修改后继续
    ↓
如需调整
    → 更新相关规划文档（MD文件）
    → 记录调整原因和内容
    → 继续创作
```

### 大纲执行检查清单

#### 每章创作前检查
- [ ] 是否已读取本章的详细规划？
- [ ] 是否明确本章的核心任务？
- [ ] 是否清楚本章的爽点类型和要求？
- [ ] 是否了解本章的情绪曲线设计？
- [ ] 是否知道本章涉及的关键情节？

#### 每章创作时检查
- [ ] 是否按照规划完成核心任务？
- [ ] 是否按照规划呈现爽点？
- [ ] 是否遵循规划的情绪曲线？
- [ ] 是否包含规划中的关键情节？
- [ ] 是否随意更改了规划内容？

#### 发现偏离时处理
- [ ] 立即停止创作
- [ ] 对比规划，找出偏离点
- [ ] 评估是否需要调整大纲
- [ ] 如需调整，触发 Gatekeeper 门控
- [ ] 等待用户确认后再继续

### 违规处理

**规则6违规**：
1. **擅自调整大纲**：立即恢复原文档，按原大纲重新创作
2. **未完成核心任务**：补充内容，确保核心任务完成
3. **偏离爽点设计**：调整内容，符合规划的爽点类型
4. **跳过关键情节**：补充情节，或触发门控申请调整大纲

---

## 违规处理

如违反以上规则：

1. **规则1违规**：立即创建正确文件夹结构，迁移已有文件
2. **规则2违规**：暂停创作，补全大纲，等待用户确认
3. **规则3违规**：补全封面图和推文
4. **规则4违规**：拆分章节文件，确保每章独立
5. **规则5违规**：立即将口头描述的内容文档化，保存为 Markdown 文件
6. **规则6违规**：
   - 擅自调整大纲：恢复原文档，按原大纲重新创作
   - 未完成核心任务：补充内容，确保任务完成
   - 偏离规划：调整内容，符合规划要求
7. **【新增】规则7违规**：
   - 文件名格式错误：重命名为正确格式 `第XXX章 章节名.md`
   - 缺少章节名：补充章节名后重命名
   - 序号格式错误：修正为 `第1章`、`第15章` 格式

## 规则 7：文件名规范规则

### 强制要求
**章节文件名必须是 `第XXX章 章节名.md` 格式**

### 文件名格式规范

#### 正确格式示例
- ✅ `第1章 初入传承.md`
- ✅ `第15章 突破境界.md`
- ✅ `第123章 终局之战.md`
- ✅ `第3章 宗门大比.md`

#### 错误格式示例
- ❌ `第001章.md`（缺少章节名）
- ❌ `第一章 初入传承.md`（应使用阿拉伯数字）
- ❌ `1. 初入传承.md`（格式不正确）
- ❌ `chapter1.md`（应使用中文）
- ❌ `第1章.md`（缺少章节名）
- ❌ `初入传承.md`（缺少章节序号）

### 命名规则详解

#### 1. 章节序号部分
- **格式**：`第` + 阿拉伯数字 + `章`
- **示例**：`第1章`、`第15章`、`第123章`
- **禁止**：`第一章`、`第01章`、`第001章`、`1.`

#### 2. 章节名部分
- **要求**：必须与章节内容相符
- **长度**：2-10个汉字为宜
- **内容**：概括本章核心内容或亮点
- **示例**：`初入传承`、`突破境界`、`宗门大比`

#### 3. 文件扩展名
- **必须是**：`.md`（Markdown格式）
- **禁止**：`.txt`、`.doc`、`.docx` 等其他格式

### 违规处理

**规则7违规**：
1. **文件名格式错误**：立即重命名为正确格式
2. **缺少章节名**：补充合适的章节名后重命名
3. **序号格式错误**：修正为阿拉伯数字格式
4. **扩展名错误**：转换为 Markdown 格式
