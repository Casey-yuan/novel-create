# 章节生成模板

每章按此模板生成，确保结构统一、节奏可控、上下文衔接自然。

---

## 标准章节模板（含字数和衔接）

```markdown
# 第XX章 [章节名]

## 章节信息
- **章节序号**: 第XX章
- **所属卷**: 第X卷 [卷名]
- **字数限制**: 2000-3000字（标准）
- **实际字数**: XXXX字
- **创作时间**: YYYY-MM-DD HH:MM

## 上下文衔接
- **承接上章**: [上一章结尾发生了什么，本章如何承接]
- **角色状态**:
  - 位置: [地点]（上章位置: [地点]）
  - 情绪: [情绪]（上章情绪: [情绪]）
  - 状态: [状态]（伤势/健康/其他）
- **悬念回应**: [如何回应上一章留下的悬念]
- **下章铺垫**: [为下一章铺垫了什么]

## 本章规划（来自卷级规划，规则6：严格执行）
- **核心任务**: [本章必须完成的核心任务] ⚠️ **必须完成，不得跳过**
- **爽点类型**: [打脸/升级/收获/反转/情感/其他] ⚠️ **必须按规划呈现**
- **爽点强度**: ⭐⭐⭐☆☆（1-5星）
- **情绪曲线**: [起-承-转-合的情绪变化] ⚠️ **必须遵循设计**
- **关键情节**: [规划中设定的关键情节1]、[关键情节2] ⚠️ **必须发生，不得更改**
- **涉及角色**: [主角]、[配角A]、[配角B]、[反派]
- **涉及地点**: [地点名称]

> **规则6提醒**：以上内容来自 `outline/卷级规划/第X卷.md`，创作时必须严格按照规划执行，不得随意调整。如需调整，必须通过 Gatekeeper 门控，经用户确认后方可修改。

---

## 正文

### 开篇（承接上文）
[承接上一章结尾，自然过渡]
[字数: 200-300字]

### 发展（情节推进）
[矛盾升级 / 信息揭示 / 力量展示]
[字数: 800-1200字]

### 高潮（爽点呈现）
[本章爽点 / 冲突爆发 / 反转发生]
[字数: 600-1000字]

### 收束（结果呈现）
[结果呈现 / 众人反应 / 章末钩子]
[字数: 300-500字]

---

## 章末信息
- **章末钩子**: [留下的悬念，引导继续阅读]
- **下章预告**: [下一章将要发生什么的简要提示]
- **章节状态**: ✓ 完成 / 待修改

## 审核记录
[如有审核，记录审核结果]

## 修改记录
[如有修改，记录修改内容]
```

## 字数控制指南

### 字数分配建议
| 章节部分 | 建议字数 | 占比 | 说明 |
|---------|---------|------|------|
| 开篇 | 200-300字 | 10-15% | 承接上文，快速入戏 |
| 发展 | 800-1200字 | 40-50% | 情节推进，信息揭示 |
| 高潮 | 600-1000字 | 30-35% | 爽点呈现，冲突爆发 |
| 收束 | 300-500字 | 15-20% | 结果呈现，章末钩子 |
| **总计** | **2000-3000字** | 100% | 标准章节 |

### 字数超标处理
如创作过程中发现字数超标：
1. **检查冗余**: 删除重复描写、过度修饰
2. **精简对话**: 合并短对话，删除无意义寒暄
3. **聚焦核心**: 保留核心情节，删减支线
4. **拆分章节**: 如情节确实丰富，考虑拆分为两章

### 字数不足处理
如字数不足：
1. **增加细节**: 环境描写、心理活动、动作细节
2. **扩展对话**: 增加对话内容，展现人物性格
3. **补充反应**: 增加配角反应、群众反应
4. **铺垫伏笔**: 增加为后续铺垫的细节

## 章节生成检查清单

每章生成后自检：

### 字数检查（强制）
- [ ] 实际字数: XXXX字
- [ ] 目标范围: 2000-3000字
- [ ] 字数状态: ✓ 符合 / ⚠ 接近上限 / ✗ 超出
- [ ] 各部分字数分配合理

### 衔接检查（强制）
- [ ] 与上一章结尾衔接自然
- [ ] 角色状态与 `CHARACTERS.md` 一致（等级、伤势、位置）
- [ ] 地点描写与 `LOCATIONS.md` 一致
- [ ] 未违反 `STORY_BIBLE.md` 中的设定规则
- [ ] 未与 `PLOT_POINTS.md` 中已有情节矛盾
- [ ] 正确回应上一章悬念
- [ ] 为下一章做好铺垫

### 内容检查
- [ ] 完成章节核心任务
- [ ] 关键情节已发生
- [ ] 爽点已呈现
- [ ] 章末有钩子

### 大纲执行检查（规则6 - 强制）
- [ ] **核心任务完成**：是否完成了规划中规定的本章核心任务？
- [ ] **爽点类型一致**：是否按照规划呈现了指定的爽点类型？
- [ ] **情绪曲线符合**：是否遵循了规划中设计的情绪曲线？
- [ ] **关键情节发生**：规划中设定的关键情节是否已发生？
- [ ] **无擅自调整**：是否擅自更改了规划中设定的内容？
- [ ] **人物状态连贯**：角色状态变化是否符合规划？

**大纲执行标准**：
- ✅ **完全符合**：100%按照规划创作，所有要求都已满足
- ⚠️ **基本符合**：90%按照规划创作，个别细节有微调（需记录）
- ❌ **不符合**：擅自调整大纲，未按规划创作（必须修改或申请调整）

**发现偏离时处理**：
1. 立即停止创作
2. 对比规划，找出偏离点
3. 评估是否需要调整大纲
4. 如需调整，必须通过 Gatekeeper 门控，经用户确认
5. 如无需调整，按原规划修改内容

### 质量检查
- [ ] 本章有明确的爽点或情节推进
- [ ] 对话有角色个性差异
- [ ] 节奏恰当，不拖沓
- [ ] 主角有可感知的成长或收获

### 记忆更新检查
- [ ] 新角色已记录到 `CHARACTERS.md`
- [ ] 角色状态变化已更新
- [ ] 新地点已记录到 `LOCATIONS.md`
- [ ] 关键情节已记录到 `PLOT_POINTS.md`
- [ ] 如有关键场景，已生成图解

## 衔接问题处理

### 常见问题及解决方案

| 问题 | 表现 | 解决方案 |
|------|------|---------|
| 时间跳跃 | 上一章是白天，本章突然是晚上 | 添加过渡描写或明确时间提示 |
| 地点突变 | 角色突然出现在新地点 | 补充移动过程的描写 |
| 状态不一致 | 上一章受伤，本章突然好了 | 补充恢复过程或说明 |
| 悬念未回应 | 上一章的悬念被忽略 | 在本章开头回应悬念 |
| 情节断层 | 两章之间缺少必要情节 | 补充过渡情节或调整规划 |

### 衔接检查流程
1. **创作前**: 阅读上一章结尾，明确承接点
2. **创作中**: 时刻关注角色状态和情节发展
3. **创作后**: 对照检查表，验证衔接自然度
4. **审核时**: Auditor 重点检查衔接问题
